# חוקי עריכת קוד ופרויקט

בכל שינוי קוד יש לבחור קודם את שיטת העריכה המתאימה לפי גודל וסוג המשימה:

## 1. שינוי קטן או מקומי
*(קובץ אחד, מספר שורות קטן, תיקון באג נקודתי, הוספת פונקציה, שינוי UI נקודתי)*
* השתמש בעריכה ישירה של הקובץ (`replace_file_content` / `multi_replace_file_content`).
* **אל תיצור סקריפט זמני.**
* ודא שהשינוי מוצג בצורה ברורה וניתנת לבדיקה דרך Git Diff.

## 2. שינוי גדול או חוזרני
*(הרבה קבצים, שינוי שמות משתנים/פונקציות בכל הפרויקט, ריפקטורינג מסיבי, מבנה תיקיות חדש, מאות החלפות)*
* העדף יצירת סקריפט Python/Bash או כלי אוטומציה אחר.
* הסקריפט חייב להיות ברור, בטוח, ולהציג בצורה מפורשת מה הוא משנה.
* **לפני הרצת סקריפט כלשהו, הסבר למשתמש מה הוא עושה וקבל את אישורו המפורש.**

## 3. במקרה של ספק
* העדף תמיד עריכה ישירה קודם.
* עבור לסקריפט רק אם העריכה הישירה תהיה ארוכה מאוד, מסוכנת או מועדת לשגיאות כתיבה/רווחים.

**המטרה:** לשמור על שקיפות ובדיקת Git קלה בשינויים קטנים, ולנצל אוטומציה רק כשיש לה יתרון אמיתי ומובהק.
