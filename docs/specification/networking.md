# תקשורת, סנכרון ופרוטוקול נתונים 🌐

הסנכרון בזמן אמת בין שלט המנחה למסך ההקרנה מתבצע ישירות מול **Firebase Realtime Database (RTDB)**. המערכת פועלת במודל Serverless מלא, כאשר הלקוחות (השלט וההקרנה) מאזינים ישירות לעדכוני המצב ומשדרים הודעות ברשת.

---

## 💾 1. מבנה הנתונים ב-Realtime Database

כל הנתונים המשויכים לחדר משחק נשמרים תחת נתיב ייחודי לפי קוד חדר בן 4 ספרות: `rooms/{ROOM_CODE}/`.
מבנה הנתונים מחולק לשלושה אזורים מרכזיים: `settings` (הגדרות החדר), `state` (מצב המשחק הפעיל) ו-`db` (מאגר השאלות והמשתתפים המלא):

## 📡 2. פרוטוקול הודעות בזמן אמת (Broadcast Messages)

בנוסף להאזנה הישירה לעץ הנתונים, המכשירים מתקשרים ביניהם באמצעות ערוץ הודעות מבוסס תור (Queue) תחת הנתיב `rooms/{ROOM_CODE}/messages`.
כל הודעה מכילה מזהה ייחודי (`id`) וחותמת זמן (`timestamp`) כדי למנוע כפילויות. טיפוסי ההודעות הנתמכים במערכת הם:

- **`STATE_CHANGED`**: עדכון מצב משחק כולל.
- **`SETTINGS_CHANGED`**: עדכון הגדרות החדר.
- **`DATABASE_SYNC`**: סנכרון מלא של בסיס הנתונים (שאלות ומשתתפים).
- **`TRIGGER_CONFETTI`**: הפעלת אפקט קונפטי במסך ההקרנה בעת הענקת נקודה.

> [!WARNING]
> **מגבלת שימוש ב-BroadcastChannel**:
> המערכת משתמשת ב-`BroadcastChannel` של הדפדפן לצורך סנכרון מהיר וחסכוני מקומי. 
> יש לשים לב כי מנגנון זה **עובד אך ורק על אותו מכשיר ואותו דפדפן** (למשל, מנחה המריץ את השלט ואת מסך ההקרנה בשתי לשוניות שונות באותו מחשב). סנכרון בין מכשירים שונים (למשל, שלט מהטלפון ומסך הקרנה מהמחשב) **אינו נתמך אופליין** ומחייב חיבור אינטרנט פעיל לענן דרך שירות Firebase Realtime Database.

## 🔌 3. זיהוי ניתוק המנחה (Presence & Disconnect handling)

מכיוון שלשלט המנחה יש סמכות בלעדית על זרימת המשחק, קטיעת הקשר שלו (למשל: עקב שיחת טלפון נכנסת, איבוד קליטת אינטרנט או סגירת הדפדפן) מחייבת השהיה מיידית של מסך ההקרנה כדי למנוע את המשך רצף המשחק ללא השגחה.

- **מימוש במערכת**: בעת טעינת שלט המנחה והחיבור ל-Firebase, מוגדר כלל האזנה לניתוק (onDisconnect handler) ברמת השרת של Firebase:
  ```typescript
  const isPausedRef = ref(rtdb, `rooms/${roomCode}/database/state/isPaused`);
  onDisconnect(isPausedRef).set(true);
  ```
- **תוצאה**: ברגע ששרת Firebase מזהה שחיבור ה-Websocket של שלט המנחה נקטע, השרת מעדכן באופן אוטומטית ויזום את הערך `isPaused` ל-`true` בבסיס הנתונים. מסך ההקרנה, שמאזין לנתיב זה, מזהה את השינוי מיידית ומציג את שכבת ההשהיה המטושטשת: *"**{שם המנחה}** יחזור בקרוב..."* עד לחיבורו המחודש של השלט.

## 🔒 4. אימות אבטחה וכניסה (Security Handshake)

למניעת כניסת שחקנים או צופים לממשק שלט המנחה ושיבוש המשחק, הוטמע מנגנון אימות מבוסס קישור:
1. בעת יצירת החדר, שם המנחה נשמר בבסיס הנתונים (`hostName`).
2. קישור שלט המנחה שנוצר (והברקוד) כולל פרמטר מפתח מיוחד ב-URL: `&host=HOST_NAME`.
3. בעת כניסה לשלט או למסך ההקרנה, הקוד בודק האם שם המנחה שב-URL (או במקרה של שלט - השם שנרשם בכניסה) תואם לשם השמור בבסיס הנתונים.
4. במידה וקיימת אי-תאימות, המשתמש מועבר למסך חסימה מאובטח ולא מתאפשרת שליטה או האזנה לנתוני החדר.
